# 미팅 기록 가이드

## 폴더 구조

미팅마다 `YYYY-MM-DD-주제` 폴더를 만들고, 아래 파일을 넣습니다.

```
2026-03-14-problem-selection/
  raw-chat.txt          ← 카톡/슬랙 채팅 내보내기 (원문 그대로)
  raw-transcript.txt    ← 줌/구글미트 트랜스크립트 (원문 그대로)
  summary.md            ← (선택) 정리된 회의록
```

## 규칙

1. **원문은 가공하지 않는다** — 복사해서 그대로 붙여넣기
2. **파일 이름은 `raw-`로 시작** — 채팅이면 `raw-chat.txt`, 트랜스크립트면 `raw-transcript.txt`
3. **정리는 안 해도 된다** — summary.md는 나중에 AI가 만들어도 됨
4. **여러 소스가 있으면 다 넣는다** — 채팅 + 트랜스크립트 둘 다 있으면 둘 다

## 왜 이렇게 하는가

- 나중에 AI가 "이 결정을 왜 했지?" 같은 질문에 답하려면 원문이 필요
- 정리하면서 빠지는 맥락(뉘앙스, 고민, 반대 의견)이 나중엔 가장 중요한 단서가 됨
- 원문이 쌓여야 RAG 도입 시 바로 활용 가능
