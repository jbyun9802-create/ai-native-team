# Growth 사고 프레임워크

> 초안은 PM이 작성하고, Growth/Marketing 담당자가 최종 컨펌합니다.

---

## 1. 성장 기회를 발견하는 방식

### 최고의 Growth 담당자가 기회를 찾을 때

- 감이 아닌 데이터에서 출발한다
- 퍼널의 가장 큰 이탈 지점을 먼저 본다
- "왜 이탈하는가?"를 정성적으로 이해한 후 정량으로 검증한다

### 사고 단계

```
Step 1: 퍼널 현황 파악 (AARRR)
  - Acquisition: 유저가 어디서 오는가?
  - Activation: 첫 경험이 "아하!"로 이어지는가?
  - Retention: 다시 오는가?
  - Revenue: 돈을 내는가?
  - Referral: 추천하는가?

Step 2: 가장 큰 이탈 지점 특정
  - 퍼널에서 가장 큰 드롭은 어디인가?
  - 그 드롭의 크기는? (절대치 + 비율)
  - 같은 단계에서 이탈하는 유저의 공통점은?

Step 3: "왜?"
  - 이탈하는 유저에게 직접 물어봤는가?
  - 정량 데이터와 정성 데이터가 같은 이야기를 하는가?
  - 경쟁사에서는 이 단계를 어떻게 풀었는가?

Step 4: 기회 크기 추정
  - 이 이탈을 X% 줄이면 전체에 어떤 영향?
  - 투자 대비 기대 효과
```

→ **STOP 룰 후보**: 시책 요청 시 "어떤 퍼널 단계의 어떤 이탈을 줄이는 것인가?" 먼저

### 반드시 사람이 확인할 것
- 데이터의 정확성 — 추적이 제대로 되고 있는가?
- 이탈 원인의 진짜 이유 — 데이터에 안 잡히는 맥락

---

## 2. 실험을 설계하는 방식

### 최고의 Growth 담당자가 실험할 때

- 모든 시책을 실험으로 접근한다 — "이게 될 거야"가 아닌 "이게 되는지 검증하자"
- 가설을 먼저 세우고, 실패 기준도 먼저 정한다
- 가장 빠르게 검증할 수 있는 방법을 찾는다

### 사고 단계

```
Step 1: 가설 구조화
  "If we [행동], then [기대 결과], because [근거]"
  예: "If we 온보딩에 프로그레스 바를 추가하면, 
       then 활성화율이 15% 증가할 것이다, 
       because 유저가 완료까지 얼마나 남았는지 알면 이탈이 줄어드니까"

Step 2: 실험 설계
  - 성공 지표: 뭘 측정할 것인가?
  - 실패 기준: 어떤 숫자 미만이면 실패?
  - 기간: 얼마나 돌릴 것인가?
  - 트래픽: 몇 %에게 노출?

Step 3: ICE 스코어링
  - Impact: 성공하면 얼마나 큰 영향?
  - Confidence: 성공할 확률은?
  - Ease: 얼마나 빠르게/쉽게 할 수 있는가?
  - → 높은 ICE부터 실행

Step 4: 결과 분석
  - 통계적으로 유의한가?
  - 성공이면: 스케일업 방법은?
  - 실패면: 왜? 가설의 어느 부분이 틀렸는가?
```

→ **THINK 룰 후보**: 실험 설계 시 가설 → 성공/실패 기준 → ICE 순서

### 반드시 사람이 확인할 것
- 가설의 "because" 부분 — AI가 그럴듯한 근거를 만들지만 실제 유저 행동과 다를 수 있음
- 실패 기준의 적절성 — 너무 낮으면 의미 없는 실험, 너무 높으면 좋은 시책도 죽임

## 3. 카피/메시지를 쓰는 방식

### 사고 단계

```
Step 1: 누구에게, 어떤 상황에서?
  - 이 메시지를 보는 유저는 누구인가?
  - 어떤 감정 상태인가? (탐색? 급함? 불안?)
  - 무엇을 기대하고 있는가?

Step 2: 핵심 메시지
  - 한 문장으로 유저에게 전달할 가치는?
  - PAS: Problem(문제) → Agitate(악화) → Solution(해결)
  - 또는: 결과 먼저 → 이유 → 행동 촉구

Step 3: 톤
  - 브랜드 보이스와 일치하는가?
  - 이 맥락에 맞는 톤인가? (친근? 전문적? 긴급?)

Step 4: 테스트
  - A/B 테스트할 수 있는 형태로 작성
  - 최소 2개 버전
```

### 반드시 사람이 확인할 것
- 브랜드 톤 일관성 — AI가 각 카피는 잘 쓰지만 전체 브랜드 톤이 흔들릴 수 있음
- 유저 감정 맥락 — "이 순간 유저가 이 메시지를 보면 어떤 기분일까?"

## 4. 채널 전략을 세우는 방식

### 사고 단계

```
Step 1: 타겟 유저가 어디에 있는가?
  - 어떤 채널을 쓰는가?
  - 어떤 콘텐츠를 소비하는가?
  - 어떤 커뮤니티에 속해있는가?

Step 2: Bullseye Framework
  - 가능한 모든 채널 나열 (20+)
  - 가장 가능성 높은 3개 선택
  - 각각 저비용 테스트 설계
  - 1~2개가 검증되면 집중

Step 3: 채널별 ROI 추적
  - CAC (고객 획득 비용) 채널별 비교
  - LTV/CAC 비율
  - 스케일 가능성
```

### 반드시 사람이 확인할 것
- 채널의 현실적 접근 가능성 — AI가 추천하는 채널이 실제로 접근 가능한지
- 경쟁사의 채널 전략과의 차별화

## 5. 지표를 관리하는 방식

### 사고 단계

```
Step 1: North Star Metric 정의
  - 유저에게 핵심 가치를 전달하는 단일 지표는?
  - 이 지표가 올라가면 비즈니스가 성장하는가?
  - 팀 전체가 이해하고 영향을 줄 수 있는가?

Step 2: 선행 지표 vs 후행 지표
  - 후행: 매출, MAU → 결과 확인 (느림)
  - 선행: 활성화율, 기능 사용률 → 미리 예측 (빠름)
  - 선행 지표를 모니터링하고, 후행 지표로 검증

Step 3: 이상 감지
  - 지표가 평소와 다르면 → 원인 분석
  - 외부 요인? (시즌, 뉴스)
  - 내부 요인? (배포, 기능 변경)
```

### 반드시 사람이 확인할 것
- North Star의 적절성 — 6개월마다 재검토
- 허영 지표 구분 — "올라가지만 비즈니스에 의미 없는 숫자"

## 6. 핸드오프하는 방식

```
Growth → PM:
  - 실험 결과 + 인사이트
  - 퍼널 데이터 + 이탈 원인 분석
  - "이걸 제품으로 만들면 효과 추정"

Growth → Design:
  - 유저가 이탈하는 구체적 순간/화면
  - 경쟁사가 이 단계를 어떻게 풀고 있는지
  - 테스트할 카피/메시지 후보

Growth → Engineering:
  - 추적이 필요한 이벤트 목록
  - A/B 테스트 기술 요구사항
  - 성능 임팩트 예상
```

## 7. AI를 쓰는 방식

### AI에게 맡겨도 되는 것
- 카피 초안 (여러 버전)
- 경쟁사 분석 초안
- 데이터 정리/시각화
- SEO 키워드 분석
- 이메일 시퀀스 초안
- 랜딩페이지 초안

### 반드시 사람이 해야 하는 것
- 퍼널 이탈 원인의 "왜?" 해석
- 실험 가설의 "because" 검증
- 브랜드 톤 최종 판단
- North Star Metric 정의
- 채널 전략의 현실적 판단
- 성공/실패 기준 설정
