Main 1 · Sub 1-1

Main 1 · Sub 1-2
결과물만 넘어가고 판단 근거가 없다
매일 발생 전파력 높음 지금 피 남
Priority 2 — 만성 비효율
Main 1 · Sub 1-3
팀 지식으로 누적되지 않는다
전파력 중간 만성 비효율
Main 2 · Sub 2-2
AI 아웃풋 신뢰도 식별 불가
전파력 중간 만성 비효율
Main 2 · Sub 2-3
팀원마다 AI 활용 수준 편차 발생
전파력 낮음 만성 비효율
Priority 3 — Future risk
Main 2 · Sub 2-1
프롬프트 기록 방법이 없다
전파력 높음 Future risk
Main 2 · Sub 2-3~4
Agent 컨텍스트 전달 및 오류 검수 체계 없음
전파력 매우 높음 Future risk
의존성 지도
컨텍스트 저장 + 판단 근거 붙이기
→
신뢰도 식별 가능
→
팀 지식 누적
컨텍스트 저장 + 판단 근거 붙이기
→
Agent 컨텍스트 전달 설계
→
Agent 검수 체계
MVP: "판단이 붙은 핸드오프" 구조 만들기
AI로 무언가를 만들었을 때, 결과물 + 왜 이렇게 했는지 (고민, 선택지, 기각 이유) 가 함께 저장된다
다음 사람이 받을 때, 결과물만 보는 게 아니라 그 판단의 맥락도 같이 받는다
저장 방식은 최소한으로 — 별도 문서 작성 없이, AI 대화에서 자동 또는 아주 가벼운 트리거로 추출된다
MVP 스콥 밖 (v2 이후)
✕ 팀 지식 베이스 누적 및 검색 — 인프라 먼저 검증 후
✕ AI 아웃풋 신뢰도 레이블링 시스템
✕ Agent 워크플로우 설계 — 사람 핸드오프 구조 먼저
