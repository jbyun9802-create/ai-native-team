# 문제 분석 및 우선순위

## 문제 정의

### Main 1: AI 아웃풋이 개인 단위에서 끝난다

AI가 만든 결과물이 팀의 자산이 아니라 개인의 채팅창에 갇혀 있다. 넘겨지는 건 "결과물"뿐이고, 그걸 만들기까지의 과정과 판단이 사라진다.

| Sub | 문제 | 설명 |
|-----|------|------|
| 1-1 | 결과물만 공유되고 "왜"가 없다 | 결과물은 전달되지만 판단 근거(고민, 선택지, 기각 이유)가 빠진다 |
| 1-2 | 다음 사람이 컨텍스트를 처음부터 다시 만들어야 한다 | 이전 작업자의 맥락을 모르니 같은 탐색을 반복한다 |
| 1-3 | 개인 AI 대화가 팀 지식으로 누적되지 않는다 | 각자의 대화창에서 쌓인 인사이트가 팀에 공유되지 않는다 |

### Main 2: 협업 흐름이 AI Agent를 참여자로 가정하지 않는다

기존 핸드오프 방식(문서, 슬랙, 회의)은 사람→사람 설명 구조였다. AI가 중간에 낀 지금은 그 구조가 맞지 않는데, 새 구조를 아무도 만들지 않았다.

| Sub | 문제 | 설명 |
|-----|------|------|
| 2-1 | 프롬프트/판단 기록 방법이 없다 | 어떤 프롬프트로 무엇을 만들었는지 추적 불가 |
| 2-2 | AI 아웃풋의 신뢰도가 불명확한 채로 넘어간다 | Agent 초안인지 사람이 검토한 산출물인지 식별 안 됨 |
| 2-3 | Agent가 컨텍스트를 받는 방식이 설계되어 있지 않다 | 핸드오프가 사람→사람 기준이라 Agent한테 뭘 어떻게 넘겨야 하는지 모름 |
| 2-4 | Agent가 틀렸을 때 누가 잡는지 정해져 있지 않다 | 사람 실수는 리뷰 프로세스가 있지만, Agent 오류 검수 체계는 없음 |
| 2-5 | 팀원마다 AI 활용 수준이 달라서 결과물 품질 편차가 생긴다 | 같은 태스크도 누가 하느냐에 따라 AI 활용도와 결과물 품질이 크게 달라짐 |

### 두 문제의 관계
- **Main 1**은 "정보 손실" 문제
- **Main 2**는 "프로세스 부재" 문제
- 도구를 만들어도 프로세스가 없으면 안 쓰고, 프로세스를 만들어도 정보가 유실되면 소용없다. **둘을 같이 풀어야 한다.**

## 우선순위 프레임워크

### 평가 축
1. **발생 빈도**: 매일 일어나는가, 특정 순간에만 발생하는가
2. **전파력**: 이 문제가 해결 안 되면 다른 문제들이 연쇄적으로 막히는가
3. **시점**: 지금 당장 고통인가, 미래의 리스크인가

### 문제별 평가

| 문제 | 빈도 | 전파력 | 시점 | 우선순위 |
|------|------|--------|------|----------|
| 컨텍스트가 채팅창에서 사라짐 (1-1) | 매일 | 높음 | 지금 피 남 | **P1** |
| 판단 근거 없이 결과물만 넘어감 (1-2) | 매일 | 높음 | 지금 피 남 | **P1** |
| 팀 지식으로 누적 안 됨 (1-3) | 자주 | 중간 | 만성 비효율 | P2 |
| AI 아웃풋 신뢰도 식별 불가 (2-2) | 자주 | 중간 | 만성 비효율 | P2 |
| 팀원별 AI 활용 수준 편차 (2-5) | 자주 | 낮음 | 만성 비효율 | P2 |
| 프롬프트 기록 방법 없음 (2-1) | 낮음 | 높음 | Future risk | P3 |
| Agent 컨텍스트 전달 미설계 (2-3) | 낮음 | 매우 높음 | Future risk | P3 |
| Agent 오류 검수 체계 없음 (2-4) | 낮음 | 높음 | Future risk | P3 |

## 의존성 지도

```
컨텍스트 저장 + 판단 근거 붙이기
  ├── → 신뢰도 식별 가능 → 팀 지식 누적
  └── → Agent 컨텍스트 전달 설계 → Agent 검수 체계
```

모든 문제의 뿌리가 하나로 수렴한다: **"판단 근거가 같이 저장되지 않는다"**

## MVP 스코프

### MVP 정의: "판단이 붙은 핸드오프" 구조

- **AI로 무언가를 만들었을 때**, 결과물 + 왜 이렇게 했는지(고민, 선택지, 기각 이유)가 함께 저장된다
- **다음 사람이 받을 때**, 결과물만 보는 게 아니라 그 판단의 맥락도 같이 받는다
- **저장 방식은 최소한으로** — 별도 문서 작성 없이, AI 대화에서 자동 또는 아주 가벼운 트리거로 추출된다

### MVP 밖 (v2 이후)
- 팀 지식 베이스 누적 및 검색 — 인프라 먼저 검증 후
- AI 아웃풋 신뢰도 레이블링 시스템
- Agent 워크플로우 설계 — 사람 핸드오프 구조 먼저

### 왜 이 MVP인가?
- **인프라 먼저**: 경험을 아무리 잘 설계해도 데이터가 없으면 의미 없음
- **의존성 뿌리**: 이게 해결되어야 나머지를 풀 수 있음
- **즉시 효과**: 지금 당장 피 나는 문제를 멈춤
