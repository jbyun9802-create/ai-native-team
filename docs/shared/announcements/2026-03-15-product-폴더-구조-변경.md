`docs/product/` 폴더가 단계별로 재정리되었습니다.

## 변경 사항

**새 구조:**
```
docs/product/
├── planning/        기획 (PRD, 문제 분석, 유저 스토리)
├── design/          디자인 (와이어프레임, 목업)
├── specs/           개발 명세 (API, 데이터 모델)
├── validation/      검증 (테스트, 피드백)
└── diagrams/        다이어그램 (기존 유지)
```

**경로 변경:**
- `problem-analysis.md` → `planning/problem-analysis.md`로 이동

## 앞으로 해주실 것
- product 관련 문서는 해당 단계 폴더에 넣어주세요
- 메타데이터에 `phase` 필드를 추가해주세요 (값: `planning` / `design` / `spec` / `validation`)
- 자세한 가이드는 `docs/product/README.md`를 참고하세요
