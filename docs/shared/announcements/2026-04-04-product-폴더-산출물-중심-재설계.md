`docs/shared/product/` 폴더 구조가 기존 "단계(phase)별"에서 **"산출물 종류별"**로 변경되었습니다.

## 변경 사항

**새 구조:**
```
docs/shared/product/
├── prd/            PRD, 문제 분석, 유저 스토리
├── specs/          기능 스펙, API 스펙, 데이터 모델
├── storybook/      컴포넌트 문서, 디자인 시스템 연동
├── research/       시장조사, 경쟁분석
├── assets/         와이어프레임, 목업, 다이어그램
└── validation/     QA 리포트, 베타 피드백
```

**삭제된 폴더:** `planning/`, `design/`, `diagrams/`

**파일 이동:**
- `planning/problem-analysis.md` → `prd/problem-analysis.md`
- `diagrams/team-workflow.html` → `assets/team-workflow.html`

**메타데이터 변경:**
- `phase` 필드 → `category` 필드로 변경
- 허용값: `prd` | `spec` | `storybook` | `research` | `validation`

## 앞으로 해주실 것
- product 관련 문서는 산출물 종류에 맞는 폴더에 넣어주세요
- 메타데이터에 `category` 필드를 사용해주세요 (기존 `phase` 대신)
- 자세한 가이드는 `docs/shared/product/README.md`를 참고하세요
