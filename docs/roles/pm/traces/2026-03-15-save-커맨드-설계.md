# 팀 작업 기록 시스템 설계 — `/save` 슬래시 커맨드

## 날짜
2026-03-15

## 목적
각 역할별 폴더에 작업 기록을 쉽게 저장할 수 있는 자동화된 방법(슬래시 커맨드) 만들기

## 핵심 내용
- 역할별 `docs/roles/{역할}/traces/` 폴더에 작업 기록을 남기는 구조 확인
- 처음엔 `save-decision`과 `save-trace`를 분리했으나, 불필요한 구분이라 `/save` 하나로 통합
- `.claude/skills/` 형식으로 슬래시 커맨드 생성
- VSCode 환경에서 커맨드 인식 안 되는 문제 발생 → 해결됨

## 결정사항
- 기록 저장 커맨드를 하나(`/save`)로 통합. 이유: 의사결정이든 대화 기록이든 "작업 후 기록을 남긴다"는 행위는 같고, 내용의 무게만 다를 뿐이므로
- 기록 파일 형식: `{날짜}-{주제}.md`, 역할별 traces 폴더에 저장

## 인사이트
- 슬래시 커맨드는 `.claude/skills/` 폴더에 `SKILL.md` 파일로 만들 수 있음

## 다른 팀원이 알아야 할 것
- `/save` 커맨드를 사용하면 작업 후 기록을 자동으로 정리해서 저장할 수 있음
- 각자 역할을 선택하면 해당 역할 폴더에 기록이 생성됨
