# PRD-2 TeamBrain Viz — 프로토타입 탐색에서 최종 방향 확정까지

## 날짜
2026-04-04

## 목적
PRD-2(팀 멘탈 모델 시각화)의 프로토타입을 실제로 만들어보면서, 무엇이 본질적인 솔루션인지 찾고 확정

## 핵심 내용
프로토타입을 6번 이상 방향 전환하면서 본질을 찾아감:

1. 3D 뇌 구조 (Three.js) → 멋있지만 매일 쓰는 도구로는 과함. 기각
2. HTML 대시보드 v1 (People + Topics + Flow 3뷰) → Topics 뷰 빼고 People + Flow만
3. HTML 대시보드 v2 (사람 탭 + reasoning chain) → Focus Areas 태그 바차트가 무의미. 태깅 부실하면 repo-structure 3 / handoff 2 같은 쓸모없는 결과
4. "지금 머릿속" 섹션 → 태그 카운트 대신 최근 traces의 topic + insight를 자연어로 표시. 기간 탭(1일/7일/30일/전체)
5. CLI (brain.py) → 터미널에서 바로 보는 정적 출력. team.zip에 포함 배포 가능
6. TUI (brain-tui.py) → 인터랙티브. 사람 전환, 기간 필터, Claude API 의미 검색
7. brain.py --html → CLI가 traces를 JSON으로 뽑아 HTML 대시보드 자동 생성
8. 최종: 뇌 지도 (Claude API 분석) → "지금 머릿속"을 trace 나열이 아니라 이 사람의 사고 에너지 분배로. Claude가 traces를 읽고 3~5개 고민 주제를 클러스터링 + 비중(%) 산출. 선택하면 시간에 따른 고민의 변화를 보여줌

## 결정사항
- 3D 뇌 시각화 기각: 실용성 없음. 매일 여는 도구가 아님
- 태그 기반 Focus Areas 기각: 메타데이터 태깅이 부실하면 의미 없는 결과
- Textual TUI 유지: VS Code 터미널에서 인터랙티브하게 사용 가능
- CLI + HTML 병행: brain.py는 빠른 조회, --html은 비주얼 대시보드
- team.zip에 brain.py + /save 동봉: Assessment 완료 후 traces 도구 세트로 배포
- "지금 머릿속"을 Claude API로 분석: trace 나열이 아닌 사고 에너지 분배 지도
- 고민 drill-down에 시간별 변화: 선택하면 그 고민이 어떻게 진화했는지 보여줌
- 의미 검색에 Claude Haiku 사용: 텍스트 매칭 아닌 맥락 기반 + excerpt 반환
- 날짜 기준을 최신 trace로: datetime.now()가 2025년이라 2026 데이터가 안 보이는 문제 해결
- 하드코딩 제거 — role 기반 자동 감지: 어떤 팀이든 /save로 traces 쌓으면 바로 동작

## 인사이트
- "뭘 빼야 하는지"가 가장 중요한 결정이었음 — 3D, Topics 뷰, 태그 바차트, 토픽 중심 뷰를 빼면서 본질에 가까워짐
- "조직의 멘탈 모델"의 본질은 fancy한 시각화가 아니라 "이 사람이 뭘 고민하고 있는지"를 바로 보는 것
- TUI에서 가장 가치 있는 기능은 "뇌 지도" + "고민 변화 추적" + "의미 검색" — 셋 다 Claude API 기반
- 프로토타입을 실제로 만들고 → 보고 → "이건 아닌데" → 다시 만드는 과정이 PRD 문서만 쓰는 것보다 훨씬 빠르게 방향을 찾음
- /save가 만드는 YAML frontmatter(date, role, topic)가 시각화에 필요한 모든 메타데이터를 이미 갖고 있음
