# 4/4 미팅 분석 — zip 배포에서 GitHub 레포 연동으로 전환

## 날짜
2026-04-05

## 목적
4/4 PRD proposal 미팅 raw transcript를 분석하여 zip → GitHub 레포 연동 결정 내용 파악 및 기록

## 핵심 내용

1. 미팅에서 팀원이 "왜 zip인가?" 질문 → PM이 "기술적으로 잘 몰라서 내린 결론일 수 있다"고 인정
2. 엔지니어 피드백: zip은 매번 다운/압축해제/재압축 파이프라인이 반복되어 불편. GitHub이 가장 편하고 프라이빗 레포면 보안도 괜찮다
3. 최종 합의: GitHub 레포에 URL 넣으면 collaborator 초대 → write 권한 → 폴더 직접 write하는 방식이 가장 단순
4. 기존 zip에 들어가던 것(팀 CLAUDE.md, 개인 .md, brain.py)을 깃 레포 연동으로 전환

## 결정사항

| 결정 | 이유 |
|------|------|
| zip 다운로드 → GitHub 레포 연동으로 변경 | zip은 매번 다운/압축해제 반복. 깃이 가장 편하고 브랜치로 개인/팀 관리 가능 |
| 프라이빗 레포 전제 | 보안 우려 해소. 대부분의 팀 레포가 이미 프라이빗 |
| collaborator 초대 + write 권한 방식 | Assessment 완료 후 결과물이 자동으로 팀 레포에 올라가는 구조 |

## 인사이트

- PM이 zip을 선택한 건 기술적 판단이 아닌 "가장 단순할 것"이라는 가정이었음
- 엔지니어 관점에서 zip 파이프라인은 오히려 복잡. 깃이 더 자연스러운 워크플로우
- 보안 이슈는 "프라이빗 레포"로 충분히 해소 가능하다는 합의
