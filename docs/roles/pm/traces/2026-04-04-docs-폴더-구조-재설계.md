# docs 폴더 구조 재설계 — 공통/개인 분리

## 날짜
2026-04-04

## 목적
흩어져 있던 레포 폴더 구조를 공통(shared)과 개인(roles)으로 명확히 분리하여 롤 간 협업을 쉽게 만들기

## 핵심 내용
- 현재 구조 분석: 미팅 기록 2곳 중복, product 폴더 소속 모호, 표준 문서가 여기저기 산재
- 3개 존으로 재설계: `_standards/`(팀 표준), `shared/`(공지+미팅+프로덕트), `roles/`(롤별 개인 공간)
- 12개 파일 이동 완료, 삭제 0건, 슬래시 커맨드 3개 모두 경로 변경 불필요 확인

## 결정사항
- product 문서는 `shared/product/`로 이동 — 프로덕트 문서는 특정 롤 소유가 아닌 팀 공통이므로
- 미팅 요약본을 각 미팅 원문 폴더 안의 `summary.md`로 통합 — 원문+요약을 한 곳에서 관리
- `_standards/` 디렉토리 신설 — metadata-standard, TEMPLATE-trace를 한 곳에 모음
- 빈 placeholder 폴더(.gitkeep)는 삭제하지 않고 그대로 유지

## 인사이트
- 슬래시 커맨드가 참조하는 경로(`docs/roles/`, `docs/shared/announcements/`)는 변경 없이 유지되어 호환성 문제 없음
- 미팅 날짜 불일치(시차)가 있었음 — 원문 폴더 날짜를 기준으로 통일

## 다른 팀원이 알아야 할 것
폴더 구조가 변경됨. `docs/product/` → `docs/shared/product/`, `docs/metadata-standard.md` → `docs/_standards/`, `docs/meetings/` → 각 `docs/shared/meetings/*/summary.md`로 통합. `/save`, `/announce`, `/briefing` 커맨드는 영향 없음.
