# /announce 커맨드 신설

## 날짜
2026-03-15

## 목적
/save를 거치지 않고 직접 공지를 작성하거나 특정 팀원에게 DM형 메시지를 보낼 수 있는 방법 만들기

## 핵심 내용
- 기존에는 공지를 만들려면 /save의 4단계(공지 생성)를 거쳐야 했음
- 작업 기록 없이 바로 공지만 보내고 싶은 니즈가 있었음
- `/announce` 슬래시 커맨드를 새로 생성함 (`.claude/commands/announce.md`)
- 사용법: `/announce` (대화형) 또는 `/announce engineering 에게: 내용` (한 줄로)
- 전체 팀(`all`) 또는 특정 역할에게 선택적으로 보낼 수 있음
- 미리보기 → 확인 → 파일 생성 흐름

## 결정사항
- `/announce`라는 이름으로 커맨드 생성 — /save와 역할이 명확히 분리됨
- 공지 파일 형식은 기존 announcements 형식 그대로 유지 (from/to/date/subject 메타데이터)
- 같은 날짜에 같은 주제가 있으면 뒤에 숫자를 붙이는 규칙 추가

## 인사이트
- 팀 커뮤니케이션 도구가 "기록 중심"에서 "소통 중심"으로도 확장 필요
- /save는 기록+공지, /announce는 순수 공지로 역할 분리가 깔끔함

## 다른 팀원이 알아야 할 것
`/announce` 커맨드가 새로 생겼으므로 모든 역할이 사용 가능. `/briefing`으로 확인 가능.
