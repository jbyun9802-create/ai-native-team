# trace 템플릿의 reasoning 갭 발견

## 날짜
2026-03-26

## 목적
어제(03-25) trace 저장 과정에서 발견한 reasoning 캡처 실패를 추적하고, 근본 원인이 템플릿 구조에 있음을 확인. 수정안 제안.

## 내가 내린 판단과 그 이유

### 판단 1: trace 첫 버전이 changelog였던 것은 템플릿의 구조적 문제

어제 /save로 trace를 생성했는데, reasoning이 빠진 changelog가 나옴. 처음에는 AI가 잘못 정리한 것으로 보였으나, PM이 만든 TEMPLATE-trace.md를 다시 확인한 결과 구조적 문제를 확인.

템플릿에 "왜"라는 단어가 있긴 함:
- `결정사항 — 무엇을, 왜`
- `생각이 바뀐 부분이 있다면 무엇이 왜`

그러나 bullet point 한 줄 구조로 되어 있어서, 실제로 채우면:
```
## 결정사항
- MVP를 Git+MD 기반으로 가지 않기로 함. 도구 편향 때문.
```
이렇게 끝남. **왜 도구 편향이라고 판단했는지, 어떤 데이터를 봤는지, 다른 선택지는 무엇이었는지 — 전부 누락.**

### 판단 2: 템플릿의 구조적 문제 3가지 식별

1. **"핵심 내용"이 reasoning보다 먼저 오고 가장 큰 섹션** → 자연스럽게 "무엇을 했는지" 위주로 채우게 유도
2. **"결정사항"이 bullet point 한 줄 구조** → "무엇을, 왜"라고 해도 한 문장으로 축약됨
3. **판단의 과정을 담을 공간 자체가 없음** → 어떤 선택지가 있었고, 왜 이것을 골랐고, 무엇을 기각했는지 쓸 곳이 없음

/save 스킬도 이 템플릿을 따르므로 동일한 문제 발생. AI가 이 템플릿 구조대로 생성하면 reasoning이 구조적으로 누락됨.

### 판단 3: AI의 한계와 인간 검수의 중요성을 보여주는 사례

AI(Claude)가 trace를 생성했을 때 reasoning이 빠진 것은, AI가 "못한" 것이 아니라 **템플릿이 그렇게 유도한 것**. 그러나 동시에 — AI가 스스로 "이 템플릿 구조가 문제입니다"라고 지적하지는 않았음. 리서처가 "내 reasoning은 어디 갔는지"를 물어본 후에야 문제가 드러남.

이것이 중요한 이유:
- AI는 주어진 구조 안에서 최선을 다함. 구조 자체를 의심하지는 않음.
- **인간이 "뭔가 빠졌는데?"를 감지하는 능력은 AI가 대체할 수 없음.**
- 우석님이 인터뷰에서 말한 "AI는 How를 해결해주고, 인간은 What을 끊임없이 고민해야 한다"와 정확히 같은 맥락
- 우석님의 AI 실패 모드 분류 중 "실수"에 해당 — 환각은 아니지만 단순히 빠트림. 다만 이 경우 빠트린 것은 AI가 아니라 템플릿 설계.

**리서치 인사이트:** AI-네이티브 팀에서 인간의 가장 중요한 역할 중 하나는 AI 산출물에서 "빠진 것"을 감지하는 것. 있는 것의 품질을 판단하는 것보다 **없는 것을 알아채는 것**이 더 어렵고 더 중요.

## 템플릿 수정 제안 (PM에게)

### 현재 (TEMPLATE-trace.md)
```markdown
## 핵심 내용
- 중요한 내용만 정리 (전체 대화 복사 금지)
- 판단에 영향을 준 부분 위주로

## 결정사항
- 내린 결정이 있다면: 무엇을, 왜
- 없으면 "없음"

## 인사이트
- 새로 알게 된 것
- 생각이 바뀐 부분이 있다면 무엇이 왜
```

### 제안 수정
```markdown
## 내가 내린 판단과 그 이유
- 어떤 선택지가 있었는지
- 왜 이것을 골랐는지 (근거가 된 데이터, 경험, 직감)
- 무엇을 기각했는지, 왜 기각했는지
- (판단별로 소제목 나눠서 작성 가능)

## 핵심 내용
- 위 판단을 뒷받침하는 맥락만 정리
- 전체 대화 복사 금지

## 알게 된 것
- 새로 알게 된 것
- 생각이 바뀐 부분: 무엇이, 왜
```

### 수정 의도
- **"판단과 그 이유"를 첫 번째 섹션으로** → 가장 먼저 채우게 유도
- **선택지 + 기각 이유까지 명시적으로 요청** → "무엇을, 왜" 한 줄로 끝나지 않도록
- **"핵심 내용"을 판단 뒤로** → 판단의 뒷받침 역할로 위치 조정
- **"결정사항"을 별도 섹션에서 제거** → "판단과 그 이유"에 통합. 분리되어 있으면 결정은 적는데 reasoning은 안 적게 됨

### /save 스킬도 함께 수정 필요
/save가 이 템플릿을 따르므로, 템플릿만 변경하면 /save 출력도 함께 변경됨. 별도 작업 불필요.

## 다른 팀원이 알아야 할 것

1. **PM에게:** TEMPLATE-trace.md 수정 제안. 위 before/after 참고. reasoning 캡처가 현재 템플릿 구조에서 구조적으로 누락되고 있음.
2. **전원에게:** AI가 생성한 문서를 받았을 때 "빠진 것이 무엇인지"를 의식적으로 확인하는 습관 필요. AI는 있는 것은 잘 정리하지만 없는 것을 스스로 알아채지 못함.
3. **웨비나 소재:** 이 에피소드 자체가 "AI-네이티브 팀에서 인간의 역할" 섹션에 활용할 수 있는 실증 사례.
