### 선정된 문제
2번과 5번은 같은 문제의 다른 면 — 합쳐서 하나의 프로덕트로 해결하기로 결정.

> "개인이 AI를 쓰는 방법은 알지만, AI가 만든 것을 어떻게 다음으로 넘길지를 설계하는 방법을 모른다. 팀원들이 각자 AI와 대화하는데 그 컨텍스트가 개인 채팅창에서 사라진다. 저 사람의 고민이 뭔지, 우선순위가 뭔지, 왜 저런 선택을 했는지 모르겠다."

## 다음 스텝

### 팀원 인터뷰 (담당: 아롬)

**목적**: 문제 분석에서 도출한 7개 서브 문제(1-1 ~ 2-4)가 실제 팀원들의 체감과 맞는지 검증하고, 문서에서 드러나지 않은 맥락을 수집한다.

**알고 싶은 것**:

1. **체감 우선순위**: 팀에서 정한 우선순위(P1/P2/P3)와 각 팀원이 느끼는 고통의 순서가 같은지, 다르다면 왜 다른지
2. **구체적 경험**: 각 문제를 실제로 겪은 상황이 있는지 — 언제, 어떤 작업에서, 어떤 식으로 문제가 됐는지
3. **현재 우회 방법**: 지금은 이 문제를 어떻게 넘기고 있는지 (예: 슬랙에 스크린샷 붙이기, 구두 설명 등) — 이게 MVP 설계의 힌트가 된다
4. **빠진 문제**: 우리가 놓친 문제가 있는지 — 목록에 없지만 실제로 불편한 것
5. **MVP 기대**: "판단이 붙은 핸드오프"라는 방향에 대해 어떤 형태면 실제로 쓸 것 같은지, 어떤 형태면 안 쓸 것 같은지
