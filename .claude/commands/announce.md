# 공지 작성

팀 전체 또는 특정 역할에게 공지를 보냅니다. DM 보내듯이 간단하게.

## 입력

$ARGUMENTS

## 사용법

```
/announce @engineering API 스펙 변경됐으니 확인해주세요
/announce @all product 폴더 구조가 바뀌었습니다
/announce @design @uxr 유저 테스트 결과 공유합니다. 첨부 문서 확인해주세요.
```

- `@all` — 전체 팀
- `@pm` `@engineering` `@design` `@uxr` `@gtm` — 특정 역할 (복수 가능)
- `@` 뒤의 나머지 텍스트가 공지 내용

## 실행 순서

### 1단계: 파싱

`$ARGUMENTS`에서 다음을 추출합니다:
- **대상**: `@` 뒤에 오는 역할명들 (all, pm, engineering, design, uxr, gtm)
- **내용**: 역할 멘션을 제외한 나머지 텍스트 전체

`$ARGUMENTS`가 비어있거나 `@`가 없으면 사용자에게 사용법을 안내하고 종료합니다.

### 2단계: 보내는 사람 확인

memory에서 사용자의 역할 정보를 확인합니다. 역할을 알 수 없으면 한 번만 물어봅니다.

### 3단계: 파일 생성

**질문 없이 바로** 공지 파일을 생성합니다:
- 경로: `docs/shared/announcements/{오늘날짜}-{주제}.md`
- 날짜 형식: `YYYY-MM-DD`
- 주제는 내용에서 핵심 키워드를 추출하여 한글로, 공백 대신 하이픈 사용
- 같은 날짜에 같은 주제가 있으면 뒤에 숫자를 붙입니다 (예: `-2`)

파일 형식:

```markdown
---
from: {보내는 역할}
to: {all 또는 [역할1, 역할2]}
date: {오늘 날짜}
subject: {내용에서 추출한 한 줄 제목}
---

{공지 내용 — 사용자가 입력한 그대로}
```

### 4단계: 완료 메시지

간단하게 확인:

```
✅ @{대상}에게 공지 전송 완료
```
