# 아침 브리핑

팀원이 하루를 시작할 때 나에게 관련된 업데이트를 확인합니다.

## 입력

사용자가 `/briefing` 뒤에 텍스트를 입력하면 추가 지시로 사용합니다: $ARGUMENTS

## 실행 순서

### 1단계: 역할 확인
사용자에게 어떤 역할로 브리핑을 받을지 물어봅니다:
- `pm` — PM (프로덕트 매니지먼트)
- `engineering` — 엔지니어링
- `design` — 디자인
- `uxr` — UX 리서치
- `gtm` — GTM (Go-To-Market)

### 2단계: 공지 확인
`docs/shared/announcements/` 폴더의 모든 파일을 읽습니다.
각 파일의 frontmatter에서 `to` 필드를 확인하여:
- `to: all`이거나
- 사용자의 역할이 `to` 목록에 포함된 공지만 필터링합니다.

### 3단계: 팀 활동 확인
각 역할별 `docs/roles/*/traces/` 폴더에서 최근 파일들을 확인합니다.
- 최근 3일 이내의 trace 파일들을 읽습니다.
- 각 trace에서 "다른 팀원이 알아야 할 것" 섹션을 확인합니다.

### 4단계: 브리핑 출력
아래 형식으로 정리해서 보여줍니다:

```
## 📬 나에게 온 공지

(공지가 있으면 각 공지의 제목, 보낸 사람, 핵심 내용을 요약)
(없으면 "새로운 공지가 없습니다")

## 📋 팀 활동 요약 (최근 3일)

### PM
- (최근 trace 요약)

### Engineering
- (최근 trace 요약)

### Design
- (최근 trace 요약)

### UX Research
- (최근 trace 요약)

### GTM
- (최근 trace 요약)

## 💡 나와 관련된 사항
(다른 팀원의 trace에서 사용자 역할에 관련된 내용이 있으면 하이라이트)
(없으면 "특별히 관련된 사항이 없습니다")
```

### 5단계: 후속 제안
브리핑 내용을 바탕으로, 오늘 우선적으로 확인하거나 대응하면 좋을 것이 있으면 제안합니다.
