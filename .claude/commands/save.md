# 작업 기록 저장

팀원이 방금 한 AI 대화에서 핵심 내용을 기록으로 저장합니다.

## 입력

사용자가 `/save` 뒤에 텍스트를 입력하면 그것을 기록 주제의 힌트로 사용합니다: $ARGUMENTS

## 실행 순서

### 1단계: 역할 확인
사용자에게 어떤 역할로 기록할지 물어봅니다:
- `pm` — PM (프로덕트 매니지먼트)
- `engineering` — 엔지니어링
- `design` — 디자인
- `uxr` — UX 리서치
- `gtm` — GTM (Go-To-Market)

### 2단계: 현재 대화에서 정보 수집
이 대화 전체를 분석해서 다음을 정리합니다:
- **제목**: 어떤 작업/주제에 대한 것인지 (간결하게)
- **목적**: 이 대화를 통해 하려 한 것
- **핵심 내용**: 대화 중 중요한 부분만 요약 (전체 복사 금지)
- **결정사항**: 내린 결정이 있다면 무엇을, 왜 그렇게 결정했는지
- **인사이트**: 새로 알게 된 것, 생각이 바뀐 부분
- **다른 팀원이 알아야 할 것**: 공유할 내용이 있는지

사용자에게 정리한 내용을 보여주고 수정할 부분이 있는지 확인합니다.

### 3단계: 파일 생성
오늘 날짜를 기반으로 파일을 생성합니다:
- 경로: `docs/roles/{역할}/traces/{오늘날짜}-{주제}.md`
- 날짜 형식: `YYYY-MM-DD`
- 주제는 한글로, 공백 대신 하이픈 사용

파일 내용은 아래 형식을 따릅니다. **반드시 YAML 메타데이터 블록을 최상단에 포함합니다:**

```markdown
---
date: {오늘 날짜}
type: trace
role: {선택한 역할}
topic: {제목 한 줄 요약}
project: ai-native-team
tags: [{핵심 키워드 3~5개, 쉼표로 구분}]
---

# {제목}

## 날짜
{오늘 날짜}

## 목적
{정리한 목적}

## 핵심 내용
{요약한 핵심 내용}

## 결정사항
{결정 내용과 이유. 없으면 "없음"}

## 인사이트
{새로 알게 된 것}

## 다른 팀원이 알아야 할 것
{공유할 내용 또는 "없음"}
```

**tags 작성 규칙:**
- 대화에서 다뤄진 핵심 개념/도구/주제를 태그로 추출
- 나중에 "이 주제에 대해 논의한 적 있어?" 검색에 걸릴 수 있는 단어로 선정
- 예: [RAG, 메타데이터, Supabase, 검색구조]

### 4단계: 공지 생성 (선택)
"다른 팀원이 알아야 할 것"이 "없음"이 아닌 경우:

1. 사용자에게 묻습니다: **"이 내용을 팀에 공지할까요?"**
2. 사용자가 공지를 원하면, 대상을 묻습니다:
   - `all` — 전체 팀
   - 특정 역할 선택 (복수 가능): `pm`, `engineering`, `design`, `uxr`, `gtm`
3. 공지 파일을 생성합니다:
   - 경로: `docs/shared/announcements/{오늘날짜}-{주제}.md`
   - 형식:

```markdown
---
from: {작성자 역할}
to: {all 또는 [역할1, 역할2]}
date: {오늘 날짜}
subject: {공지 제목}
---

{공유할 내용 — 간결하게, 팀원이 바로 이해할 수 있도록}
```

### 5단계: 완료 메시지
저장된 파일 경로를 알려줍니다. 공지를 생성한 경우, 공지 대상과 파일 경로도 함께 알려줍니다.
